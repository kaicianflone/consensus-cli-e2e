# consensus-cli-e2e

Internal E2E harness for `@consensus-tools/consensus-tools`.

## Goals

- Validate **local** consensus flow generated by `consensus-tools init` (`.consensus/api/*.sh`).
- Validate **standalone CLI** full lifecycle (`jobs`, `submissions`, `votes`, `resolve`, `result`) using npm-installed package.
- Keep this repo stable so we can rerun it on any engine/CLI change.

## Install

```bash
npm install
```

## Run tests

```bash
npm test
```

## Known note (current npm release)

The generated local `resolve.sh` template in npm `@consensus-tools/consensus-tools@0.1.4` uses a TypeScript-only declaration in a JS eval block.
This harness resolves local jobs directly through `JobEngine` for the local-script test path so CI remains green on the published package.
(Already fixed upstream in `consensus-tools` main.)

## Test against local engine changes

From this repo, switch dependency to the local checkout:

```bash
npm run deps:local
npm test
```

Switch back to npm registry package:

```bash
npm run deps:npm
npm test
```
